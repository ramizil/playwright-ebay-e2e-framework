6. Variant Selection — How All UI Patterns Are Handled

eBay product pages use three different UI patterns for variant selectors. The framework detects and handles all of them in sequence:

#UI PatternDetectionHandler MethodExample
1 Native <select> select[id^='msku-sel'], .x-msku select _select_size_if_present()
_select_remaining_variant_dropdowns()		 Size dropdown, Model dropdown
2 Swatch buttons [data-testid='x-msku'] button, .x-msku button _select_color_if_present() Colour swatches, Style tiles
3 Custom listbox
<button aria-haspopup="listbox">		 .listbox-button__control, .btn__text contains "Select" _select_remaining_listbox_buttons() Color: Select, Model: Select (headphones)

Placeholder detection: A dropdown/button needs selection if its current text contains any of: "select", "choose", "pick", "- -", "--", or its value is "", "-1", or "0".

7. Step Dependencies (auto-skip on failure)

How it works: The pytest_runtest_setup hook in conftest.py checks if any earlier step in the same class has failed. If yes, it calls pytest.skip("Previous step failed"). Tracked per parametrised variant via _failed_class_nodes on the pytest config object.

9. Retry & Error Handling

MechanismWhereRetriesBehaviour
@with_retry BasePage.type_text(), ProductPage._click_add_to_cart() 3 attempts Exponential backoff; re-locates element on each retry via SmartLocator fallback strategy
Add-to-cart retry cart_actions.py — per item loop 2 attempts Full page reload + re-select variants + re-click Add to Cart
SmartLocator fallback BasePage.find_element() N strategies Tries CSS first, falls back to XPath if CSS fails
CAPTCHA detection BasePage._check_for_captcha() — Raises CaptchaDetectedError with screenshot; test fails cleanly
Step-skip on failure conftest.py — pytest_runtest_setup — If any earlier step in the same class failed, pytest.skip()
Teardown error guard conftest.py — class_context fixture — try/except around ctx.close() to suppress TargetClosedError
