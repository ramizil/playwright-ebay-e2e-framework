Section 3 — Smart Locator Selection (בחירת לוקייטורים חכמה)

PassAt least 2 alternative locators per element

SmartLocator.__post_init__ enforces a minimum of 2 strategies at construction time. Every element across all page objects defines CSS + XPath alternatives.

core/smart_locator.py:77-83 def __post_init__(self) -> None: """Require at least two strategies.""" if len(self.strategies) < 2: raise ValueError( f"SmartLocator '{self.name}' requires at least 2 strategies, " f"got {len(self.strategies)}." )

Example element with dual locators:

pages/home_page.py:36-42 SEARCH_INPUT = SmartLocator( name="search_input", strategies=[ LocatorStrategy("css", "#gh-ac", "search input by stable ID"), LocatorStrategy("xpath", "//input[@name='_nkw' and @type='text']", "search input by name attr"), ], )
PassRuntime fallback: if primary fails, try alternative

BasePage.find_element() iterates through all strategies in order. If strategy #1 times out, it moves to strategy #2, and so on.

core/base_page.py:194-208 for idx, strategy in enumerate(smart_locator.strategies, start=1): try: locator = self._create_playwright_locator(strategy) locator.wait_for(state=state, timeout=timeout) self.logger.info( "✓ [%s] Strategy #%d succeeded: %s", smart_locator.name, idx, strategy.description or strategy.value, ) return locator except (PwTimeout, Exception) as exc: last_error = exc self.logger.warning( "✗ [%s] Strategy #%d failed (%s): %s", ... )
PassNumber of attempts = number of locators defined

The for loop in find_element() iterates over all strategies in smart_locator.strategies. An element with 3 strategies gets 3 attempts.

PassLogic in Base Page layer — tests stay clean

All locator resolution, fallback, retry, and logging lives in BasePage.find_element(). Tests call clean methods like home.search("laptop") — no locator logic visible in test code.

Section 3 — Parallel Execution (הרצות מקביליות)

PassBrowser matrix: Chrome / Firefox / Edge simultaneously

Three approaches are available:

  • Docker Compose — 3 parallel containers (Chromium, Firefox, msedge), each with isolated volumes.
  • CLI — EBAY_BROWSER=msedge pytest tests/ -v
  • Config — config.yaml → browsers: [chromium, firefox, msedge]
docker-compose.yml → 3 services: test-chromium, test-firefox, test-msedge config/settings.py:34-44 → BROWSER_PROFILE_MAP: chromium, firefox, webkit, chrome, msedge, etc.
PassSession isolation: each test in its own browser context

Every test function gets its own BrowserContext (function-scoped context fixture). Class-based flow tests share a context within the class but are isolated from other classes.

tests/conftest.py:210-224 @pytest.fixture() def context(browser, test_settings) -> Generator[BrowserContext, None, None]: """Create an isolated browser context for a single test function.""" ctx = browser.new_context(**_make_context_opts(test_settings)) ... @pytest.fixture(scope="class") def class_context(browser, test_settings) -> Generator[BrowserContext, None, None]: """Class-scoped browser context shared across all step methods."""

Context options include unique locale, timezone, and user agent — no session leakage between tests.

Requirement 4 — Resilience (התמודדות עם סביבות לא יציבות)

PassRetry with Backoff

The @with_retry decorator implements exponential backoff with jitter:

wait = backoff_factor × 2^(attempt-1) × [1 ± 0.25 jitter]

Config: max_attempts: 3, backoff_factor: 1.0, jitter: true

Used on: click(), fill(), type_text(), _click_add_to_cart()

core/retry_handler.py → @with_retry decorator config/config.yaml:17-20 → retry settings

PassGraceful Recovery

Multiple layers of graceful recovery:

  • Popup dismissal — accept_cookies_if_present(), _dismiss_shipping_dialog(), _dismiss_overlay_popup()
  • CAPTCHA detection — _check_for_captcha() fails fast with clear error
  • Optional elements — variant selectors use optional=True to skip gracefully
  • Overlay auto-dismiss on retry — _retry_callback calls _dismiss_overlay_popup() between attempts
  • Class-flow step skipping — if step N fails, steps N+1..M are auto-skipped (TestNG-style dependsOnMethods)
